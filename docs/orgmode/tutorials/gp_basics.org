* GP Basics

** Gaussian Process Regression

A GP models an unknown function as a distribution over functions, characterized
by a mean function and a kernel (covariance function). For potential energy
surfaces, the GP jointly models energies and gradients, providing both
predictions and uncertainty estimates.

Given training data ={(x_i, E_i, G_i)}= (coordinates, energy, gradient), the GP
posterior at a new point provides:

- Predicted energy and gradient (posterior mean)
- Uncertainty estimates (posterior variance)

The joint energy+gradient model is the key to reducing oracle calls: a single
energy+gradient evaluation provides =1 + D= observations (one scalar energy plus
=D= gradient components), giving the GP much more information per oracle call
than energy-only models.

** Kernels

ChemGP provides two kernel types via the =Kernel= enum.

*** MolInvDistSE (molecular systems)

Maps Cartesian coordinates to inverse interatomic distances, then applies a
squared exponential in feature space:

#+begin_example
f_ij(x) = 1 / ||x_i - x_j||
k(x, y) = sigma^2 * exp(-sum_p theta_p^2 * ||f_p(x) - f_p(y)||^2)
#+end_example

This provides rotational and translational invariance by construction, with
pair-type-specific length scales (H-H, H-C, etc.).

#+begin_src rust
use chemgp_core::kernel::{Kernel, MolInvDistSE};
let kernel = Kernel::MolInvDist(MolInvDistSE::isotropic(1.0, 1.0, vec![]));
#+end_src

The =isotropic= constructor creates a kernel with uniform length scales across
all pair types. The pair scheme is built automatically from the atomic numbers.

*** CartesianSE (arbitrary surfaces)

Operates directly on raw coordinates:

#+begin_example
k(x, y) = sigma^2 * exp(-theta^2 * ||x - y||^2)
#+end_example

Two hyperparameters: signal variance =sigma^2= and a single inverse length scale
=theta=. Use for analytical surfaces (Muller-Brown, model potentials) or systems
where rotational invariance is not needed.

#+begin_src rust
use chemgp_core::kernel::{Kernel, CartesianSE};
let kernel = Kernel::Cartesian(CartesianSE::new(100.0, 2.0));
#+end_src

The arguments are initial =sigma^2= and =theta= values; SCG training adjusts
them during optimization.

** Training via MAP-NLL

Hyperparameters (signal variance, length scales) are optimized by minimizing the
negative log-likelihood with a MAP prior, using the Scaled Conjugate Gradient
(SCG) optimizer. Data-dependent initialization (=init_kernel=) estimates starting
hyperparameters from the training data ranges.

The MAP prior prevents overfitting when training data is sparse (the typical
regime in GP-accelerated optimization, where only a handful of oracle calls have
been made). Prior variance is adaptive: pair types with fewer distances get
tighter priors, preventing length scales from wandering when constraint data is
limited.

** Kernel Blocks

For each pair of training points, the kernel computes four blocks:

- =k_ee= :: energy-energy covariance (scalar)
- =k_ef= / =k_fe= :: energy-gradient cross-covariance (D-vectors)
- =k_ff= :: gradient-gradient covariance (D x D matrix)

The full covariance for N points with D-dimensional gradients is N*(1+D) x
N*(1+D). This grows quickly, which is why FPS subset selection and RFF
approximation are essential for scalability.

** Covariance Matrix Structure

The full covariance matrix for N training points with D-dimensional gradients
has a block structure:

#+begin_example
         E_1      E_2     ...   G_1^1    G_1^2   ...   G_N^D
E_1   [ k_ee+ne  k_ee    ...   k_ef     k_ef    ...   k_ef   ]
E_2   [ k_ee     k_ee+ne ...   k_ef     k_ef    ...   k_ef   ]
...
G_1^1 [ k_fe     k_fe    ...   k_ff+ng  k_ff    ...   k_ff   ]
...
#+end_example

where =ne= is the energy noise variance plus jitter, and =ng= is the gradient
noise variance plus jitter. The matrix is symmetric by construction.

** PredModel dispatch

The =PredModel= enum in =predict.rs= dispatches between exact GP and RFF
approximation for inner-loop predictions:

#+begin_src rust
pub enum PredModel {
    Gp(GPModel),
    Rff(RffModel),
}
#+end_src

All optimizers use =build_pred_model()= to construct the appropriate variant.
When =rff_features > 0=, the RFF approximation provides O(D_rff) prediction cost
instead of O((N*(1+D))^3) for exact GP. Hyperparameters are always trained on
the exact GP (via SCG on the FPS subset); only the prediction phase uses RFF.
