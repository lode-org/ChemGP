A Julia package for Gaussian Process (GP) guided molecular geometry
optimization. ChemGP provides GP surrogate models with molecular kernels for
energy surface exploration, including minimization, saddle-point search (dimer
method), nudged elastic band (NEB) path optimization, and optimal transport GP
dimer (OTGPD).

* Features

- *Molecular kernels*: Inverse-distance based SE, Matern 5/2, and Matern 3/2
  kernels with kernel composition (sum, product, constant offset)
- *GP surrogate*: Full covariance with energy+gradient observations, automatic
  hyperparameter optimization via marginal likelihood
- *Minimization*: Trust-region GP-guided geometry optimization
- *Dimer method*: GP-accelerated saddle point search
- *NEB*: Standard NEB, GP-NEB with all-image evaluation (AIE) and
  outer-image-only evaluation (OIE)
- *OTGPD*: Optimal Transport GP Dimer for transition state search
- *Oracles*: Built-in Lennard-Jones, Muller-Brown, and LEPS potentials; RPC
  oracle for remote potential evaluation via [[https://github.com/OmniPotentRPC/rgpot][rgpot]]
- *AtomsBase integration*: Optional package extension for loading structures
  from standard file formats (extxyz, POSCAR, etc.) via AtomsBase/AtomsIO

* Installation

ChemGP.jl is not yet registered. Install directly from GitHub:

#+begin_src julia
using Pkg
Pkg.add(url="https://github.com/lode-org/ChemGP.jl")
#+end_src

* Quick Start

#+begin_src julia
using ChemGP

# Create a GP model with a molecular kernel
kernel = MolInvDistSE(1.0, 1.0)
model = GPModel(kernel; noise=1e-6)

# Set up training data for a 3-atom system
td = TrainingData(3)

# Evaluate oracle and add point
coords = random_cluster(3, 2.0)
E, G = lj_energy_gradient(coords)
add_point!(td, coords, E, G)

# Train and predict
train_model!(model, td)
E_pred, G_pred = predict(model, td, coords)
#+end_src

See the [[https://lode-org.github.io/ChemGP.jl/dev/tutorials/quickstart/][Quick Start tutorial]] for a complete walkthrough.

** Generating Figures

Tutorial figures live in =scripts/figures/tutorial/= with a two-stage
pipeline: *generators* produce HDF5 data files, *plotters* read HDF5 and
render PDFs. A Python orchestrator (=figure_runner.py=) manages the
lifecycle and provides real-time metrics via a TCP JSONL socket.

#+begin_src text
Julia gen script
  |-- TCP socket (metrics) --> figure_runner.py --> stdout + JSONL
  |-- HDF5 file (all data) --> Julia plot script --> PDF
#+end_src

Setup:

#+begin_src sh
pixi run -e figures fig-install     # Julia deps (HDF5, CairoMakie, etc.)
pixi run -e figures fig-pyinstall   # Python deps (HCN landscape only)
#+end_src

Run by tier (no RPC server needed for tiers 1-2):

#+begin_src sh
pixi run -e figures fig-tier1   # MB + LEPS basics
pixi run -e figures fig-tier2   # MB + LEPS advanced
pixi run -e figures fig-tier3   # PET-MAD + HCN (RPC required)
#+end_src

Individual figures, generation-only, and plot-only variants are also
available. See =scripts/figures/tutorial/README.md= for the full task
list, HDF5 file structure, and how to add new figures.

#+begin_export markdown
| Task | Command | Description |
|------|---------|-------------|
| Install Julia deps | `pixi r -e figures fig-install` | Figure script Julia dependencies |
| Install Python deps | `pixi r -e figures fig-pyinstall` | Python deps for HCN landscape |
| Tier 1 | `pixi r -e figures fig-tier1` | MB + LEPS basics (no RPC) |
| Tier 2 | `pixi r -e figures fig-tier2` | MB + LEPS advanced (no RPC) |
| Tier 3 | `pixi r -e figures fig-tier3` | PET-MAD + HCN (RPC required) |
#+end_export

* Contributing

1. Clone and set up the environment:
   #+begin_src sh
   git clone https://github.com/lode-org/ChemGP.jl.git
   cd ChemGP.jl
   pixi install
   pixi r instantiate
   #+end_src

2. Install pre-commit hooks:
   #+begin_src sh
   uvx pre-commit install
   #+end_src

3. Make your changes, then verify:
   #+begin_src sh
   pixi r test
   pixi r lint
   #+end_src

4. Code style is enforced by [[https://domluna.github.io/JuliaFormatter.jl/stable/][JuliaFormatter]] (Blue style,
   4-space indent). Run =pixi r fmt= to auto-format before committing.

5. Architecture diagrams live in =docs/src/assets/diagrams/= as Graphviz
   =.dot= files. After editing, regenerate with =pixi r -e docs diagrams=.
